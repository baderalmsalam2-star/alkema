تطبيق ويب (React) لإدارة الجداول الأسبوعية لنادٍ شبابي. هذا الملف هو ملف التسليم لمتابعة العمل في **Claude Code**.

3. **`README.md`** — هذا الملف (الشرح والمواصفات).

> **أقل ملف مطلوب:** `kun-mahiran-scheduler.jsx`. لكن وجود هذا الـ README مع الملف يخلّي Claude Code يفهم كل القواعد بدون شرح طويل.

## ملخص المشروع (القواعد المعتمدة)

### المراحل والمجموعات (11 مجموعة)
- **ابتدائي (5):** أبو بكر، عمر، عثمان، علي، طلحة
- **متوسط (3):** الزبير، سعد، سعيد
- **ثانوي (3):** أبو عبيدة، عبدالرحمن، أسامة بن زيد
- الثانوي يتبع **نفس قواعد المتوسط**.

### الأيام
السبت، الاثنين، الأربعاء.

### الجداول الزمنية (تختلف بين الابتدائي والمتوسط/الثانوي)
- **الابتدائي:** استقبال 3:45–4:10 · ح١ 4:10–4:50 · ح٢ 4:55–5:40 · ح٣ 5:45–6:30 · كافتيريا 6:30–6:50 · خاطرة+صلاة المغرب 6:50–7:05 · الحصة الأخيرة 7:05–8:00
- **المتوسط/الثانوي:** استقبال 3:45–4:10 · ح١ 4:10–4:50 · ح٢ 4:55–5:40 · ح٣ 5:45–6:50 · خاطرة+صلاة المغرب 6:50–7:05 · كافتيريا 7:05–7:20 · الحصة الأخيرة 7:20–8:00

### الأنشطة الإجبارية (مقفلة، تلقائية، المشرف ما يغيّرها — المنسّق فقط)
- **حلقة قرآن:** ابتدائي (السبت ح٢، الاثنين ح١) · متوسط/ثانوي (السبت ح١، الاثنين ح٢). **أسامة بدون قرآن**.
- **سباحة:** بمواعيد ثابتة لكل مجموعة (مسحوبة من جدول المسبح).
- **سكيب روم** (متوسط/ثانوي، الاثنين) و**ألعاب داخلية** (ابتدائي، السبت) — نفس النشاط باسمين، يديره صالح العربيد.
- **كرة قدم:** مقفلة إجبارية وتُوزّع تلقائياً (خوارزمية backtracking). الابتدائي **كل يوم**؛ المتوسط/الثانوي **السبت والاثنين فقط** (ممنوعة الأربعاء)، **عدا أسامة** (بدون كرة قدم).
- **خاطرة + صلاة المغرب:** إجبارية للجميع.

### المرافق والسعة (محرّك تعارض حقيقي خلف الكواليس)
17 مرفقاً: `ملعب 1، ملعب 2، صالة تحت 1، صالة تحت 2، المسبح، المصلى، الكافتيريا، B1–B6، F1–F4`.
- **المسبح يتسع لمجموعتين** (سعة = 2)؛ بقية المرافق سعة = 1.
- كرة القدم: الابتدائي → `صالة تحت 1` ثم الملاعب؛ المتوسط/الثانوي → `ملعب 1` ثم `ملعب 2`.
- النظام يمنع اختيار أي نشاط إذا كانت مرافقه ممتلئة في تلك الفترة (يظهر "ممتلئ").

### الأنشطة الاختيارية (يختارها المشرف)
تحديات، نشاط ثقافي، سينما تربوية، كروسفيت، ورش (★)، ونشاط خاص (اسم + اختيار مرفق من القائمة).

### الورش (يضبطها المنسّق)
لكل ورشة: الاسم، **الحد** (أقصى مرّات للمجموعة الواحدة)، **السعة** (أقصى عدد مجموعات بنفس الوقت)، وكود دخول.

### البدلاء
طلب بديل لأي يوم (مع سبب اختياري)؛ يُسنَد أول بديل غير مشغول.

### الأدوار والأكواد (دخول بـ 4 أرقام، تلقائي، قابل للتعديل من المنسّق)
- المنسّق: `4071` (إدارة كاملة)
- المشرفون: `2350` (كود موحّد، يختار مجموعته)
- كابتن السباحة (د. خالد): `6184` · كابتن الكرة (عبدالله السلطان): `5729` · صالح العربيد: `3846`
- الورش: ورشة التصوير `9162` · ورشة الوضوء `7503`
- الكباتن والورش يشوفون **حصصهم فقط** (المجموعة + الوقت + النشاط + المكان).

### مسار الاعتماد
draft (قيد الإعداد) → المشرف يضغط **"إرسال للاعتماد"** (يُتاح فقط لما يكتمل الجدول) → pending (بانتظار الاعتماد) → المنسّق يعتمد → approved (معتمد). أي تعديل بعد الاعتماد يرجّعه قيد الإعداد. أولياء الأمور يشوفون المعتمد فقط.

### واجهة أولياء الأمور / أفراد المجموعة (بدون دخول)
قائمة المجموعات → صفحة المجموعة فيها تبويبان: **"البرنامج"** (أنشطة بإيموجيات، بدون أوقات) و**"المواعيد والأماكن"** (الوقت + النشاط + المكان، لأفراد المجموعة وقت النشاط). تظهر بعد الاعتماد فقط.

### اكتمال الجدول
يظهر "الجدول مكتمل" فقط عند تعبئة **كل** الحصص الفاضية (لا حصة فارغة).

## الوضع الحالي والخطوة القادمة

- **الحالي:** نموذج كامل بملف واحد. التخزين `localStorage`/`window.storage` — **محلي لكل متصفّح، غير مشترك**. لذلك مسار الاعتماد لا يعمل بين أجهزة مختلفة.
- **المطلوب (الخطوة القادمة في Claude Code):**
  1. تحويله إلى مشروع **Vite + React** مرتّب (تقسيم المكوّنات، بناء إنتاجي).
  2. إضافة **باك-إند + قاعدة بيانات مشتركة** (مثل Supabase أو Netlify Functions مع قاعدة بيانات) لحفظ الجداول والاعتمادات بشكل مشترك.
  3. **دخول حقيقي** بالأكواد (مع تخزين آمن).
  4. النشر على Netlify/Vercel (الطبقة المجانية تكفي للبداية).

### ملاحظات تقنية مفيدة لـ Claude Code
- كل المنطق (القواعد، محرّك التعارض، التوزيع التلقائي للكرة، الاعتماد) موجود في `kun-mahiran-scheduler.jsx` — يُعاد استخدامه كما هو في الواجهة، ويُربط بطبقة بيانات على السيرفر بدل `store`.
- الدوال الداخلية للعرض (`Board`, `Editor`, …) تُستدعى كدوال `Editor(...)` وليست عناصر `<Editor/>` لتفادي فقدان التركيز في الحقول — يُحافَظ على هذا النمط أو يُعاد هيكلتها لمكوّنات منفصلة.
- شعار النادي حالياً مربّع فيه حرف "م" مؤقت — سيُستبدل بشعاري **الماهر** و**سالزبوري** لاحقاً.
